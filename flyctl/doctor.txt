The DOCTOR command allows you to debug your Fly environment

Usage:
  flyctl doctor [flags]
  flyctl doctor [command]

Available Commands:
  diag        Send diagnostic information about your applications back to Fly.io.

Flags:
  -a, --app string      Application name
  -c, --config string   Path to application configuration file
  -h, --help            help for doctor
  -j, --json            JSON output

Global Flags:
  -t, --access-token string   Fly API Access Token
      --verbose               verbose output

Use "flyctl doctor [command] --help" for more information about a command.
