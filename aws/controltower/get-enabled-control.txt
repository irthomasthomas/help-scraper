NAME
       get-enabled-control -

DESCRIPTION
       Provides  details  about the enabled control. For usage examples, see `
       the        AWS         Control         Tower         User         Guide
       https://docs.aws.amazon.com/controltower/latest/userguide/control-api-examples-short.html`__
       .
          Returned values

       o TargetRegions: Shows target AWS Regions where the enabled control  is
         available to be deployed.

       o StatusSummary: Provides a detailed summary of the deployment status.

       o DriftSummary: Provides a detailed summary of the drifted status.

       See also: AWS API Documentation

SYNOPSIS
            get-enabled-control
          --enabled-control-identifier <value>
          [--cli-input-json <value>]
          [--generate-cli-skeleton <value>]
          [--debug]
          [--endpoint-url <value>]
          [--no-verify-ssl]
          [--no-paginate]
          [--output <value>]
          [--query <value>]
          [--profile <value>]
          [--region <value>]
          [--version <value>]
          [--color <value>]
          [--no-sign-request]
          [--ca-bundle <value>]
          [--cli-read-timeout <value>]
          [--cli-connect-timeout <value>]

OUTPUT
       enabledControlDetails -> (structure)
          Information about the enabled control.

          arn -> (string)
              The ARN of the enabled control.

          controlIdentifier -> (string)
              The control identifier of the enabled control.  For  information
              on how to find the controlIdentifier , see the overview page .

          driftStatusSummary -> (structure)
              The drift status of the enabled control.

              driftStatus -> (string)
                 The drift status of the enabled control.

                 Valid values:

                 o DRIFTED : The enabledControl deployed in this configuration
                   doesnt match the configuration that AWS Control  Tower  ex-
                   pected.

                 o IN_SYNC : The enabledControl deployed in this configuration
                   matches the configuration that AWS Control Tower expected.

                 o NOT_CHECKING : AWS Control Tower does not check  drift  for
                   this  enabled  control. Drift is not supported for the con-
                   trol type.

                 o UNKNOWN : AWS Control Tower is not able to check the  drift
                   status for the enabled control.

          statusSummary -> (structure)
              The deployment summary of the enabled control.

              lastOperationIdentifier -> (string)
                 The last operation identifier for the enabled control.

              status -> (string)
                 The deployment status of the enabled control.

                 Valid values:

                 o SUCCEEDED  :  The enabledControl configuration was deployed
                   successfully.

                 o UNDER_CHANGE : The enabledControl configuration  is  chang-
                   ing.

                 o FAILED : The enabledControl configuration failed to deploy.

          targetIdentifier -> (string)
              The  ARN  of  the organizational unit. For information on how to
              find the targetIdentifier , see the overview page .

          targetRegions -> (list)
              Target AWS Regions for the enabled control.

              (structure)
                 An AWS Region in which AWS Control Tower expects to find  the
                 control deployed.

                 The  expected  Regions are based on the Regions that are gov-
                 erned by the landing zone. In certain cases, a control is not
                 actually  enabled  in  the Region as expected, such as during
                 drift, or mixed governance .

                 name -> (string)
                     The AWS Region name.
