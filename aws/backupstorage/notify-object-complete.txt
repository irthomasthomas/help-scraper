NAME
       notify-object-complete -

DESCRIPTION
       Complete upload

       See also: AWS API Documentation

       See 'aws help' for descriptions of global parameters.

SYNOPSIS
            notify-object-complete
          --backup-job-id <value>
          --upload-id <value>
          --object-checksum <value>
          --object-checksum-algorithm <value>
          [--metadata-string <value>]
          [--metadata-blob <value>]
          [--metadata-blob-length <value>]
          [--metadata-blob-checksum <value>]
          [--metadata-blob-checksum-algorithm <value>]
          [--cli-input-json <value>]
          [--generate-cli-skeleton <value>]

OPTIONS
       --backup-job-id (string) Backup job Id for the in-progress backup

       --upload-id (string) Upload Id for the in-progress upload

       --object-checksum (string) Object checksum

       --object-checksum-algorithm (string) Checksum algorithm
          Possible values:

          o SUMMARY

       --metadata-string (string) Optional metadata associated with an Object.
       Maximum string length is 256 bytes.

       --metadata-blob (blob) Optional metadata  associated  with  an  Object.
       Maximum length is 4MB.

       --metadata-blob-length (long) The size of MetadataBlob.

       --metadata-blob-checksum (string) Checksum of MetadataBlob.

       --metadata-blob-checksum-algorithm (string) Checksum algorithm.
          Possible values:

          o SHA256

       --cli-input-json  (string) Performs service operation based on the JSON
       string provided. The JSON string follows the format provided by  --gen-
       erate-cli-skeleton.  If  other  arguments  are  provided on the command
       line, the CLI values will override the JSON-provided values. It is  not
       possible to pass arbitrary binary values using a JSON-provided value as
       the string will be taken literally.

       --generate-cli-skeleton (string) Prints a  JSON  skeleton  to  standard
       output without sending an API request. If provided with no value or the
       value input, prints a sample input JSON that can be used as an argument
       for  --cli-input-json.  If provided with the value output, it validates
       the command inputs and returns a sample output JSON for that command.

       See 'aws help' for descriptions of global parameters.

OUTPUT
       ObjectChecksum -> (string)
          Object checksum

       ObjectChecksumAlgorithm -> (string)
          Checksum algorithm
